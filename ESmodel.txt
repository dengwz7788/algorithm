CommonJS 和 ES Module 的区别？

区别点：
    1. CommonJS 是 nodejs的环境，通过 require 方式导入，通过 export.Module 导出，导出的是引用
    2. ES Module 是 ES6的新特性，通过 import 方式导入，通过 export 导出，导出的 原始值和对象（数组，函数）

加载时机：
    1. CommonJS是同步加载模块的，它在代码执行时加载模块
    2. ES Module是异步加载，它在代码解析的时候加载模块


CommonJS的加载流程：
    1. 查找模块 - 根据路径进行查询，如是模块名，则从node_modules目录中查找
    2. 文件读取
    3. 包装模块 - 防止全局命名空间污染
    4. 执行模块代码 - 
    5. 缓存模块 - 代码执行之后会被存储到require.cache
    6. 返回导出值

ES Module的加载流程（nodejs）：
    1. 文件识别
    2. 解析和编译
    3. 加载依赖
    4. 链接和绑定
    5. 缓存模块
    6. 执行模块代码

ES Module的加载流程（浏览器）：
    1. 定义模块
    2. 导入模块
    3. 生产<script type="module" />标签
    4. 浏览器解析和执行
    4. 模块解析和执行 

静态导入和动态导入的区别：
    1. 静态导入允许编译器和游览器在加载和执行模块之前进行优化和静态分析，有助于提高性能
    2. 动态导入提供更多的灵活性和动态性。
